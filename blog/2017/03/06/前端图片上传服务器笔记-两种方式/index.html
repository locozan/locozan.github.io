前端上传图片到服务器的两种方式

前端开发中，经常要提交一些用户信息，而表单形式上传文件是最简单的，上传时经常也要把图片传给后台，在平时工作中采取的方案一般有两种：
1：把图片转为 base64 数据，然后后台保存 base64，之后前端读取的时候再解析 base64
2：把图片当文件上传，存储于后台的静态目录
这里主要也是介绍下平时自己使用的这两种方案的实现方式。

创建服务器搭建环境

如何实现浏览器向服务器提交数据并上传图片，然后怎么查看提交的是否成功？当然，这少不了要写点 Node.js 来创建一个接收我们信息的服务器。
首先你电脑得有 Node.js 跟 npm 包。然后我们在开始一步步进行以下操作：

1.创建一个文件夹： file-upload
2.在终端 ~cd 到 file-upload 文件夹下，输入命令：npm init

上传图片方法一

实现的方式是：转 base64 后返回

该方案缺点：

  1. Base64 转换后的数据理论上将要比原来的大1/3，消耗流量，浪费空间。
  2. 转化为 Base64 之后存储为图片或者字符串 会增加了解析转换的开销。

上传图片方法二

实现方式是：把图片当文件上传（multipart/form-data），保留到一个前端可以访问的位置上

最后粘贴任意一种方法然后在终端输入命令： node index.js 就会跳出 Server started! ，那么你就可以在浏览器输入地址： http://127.0.0.1:8000 然后输入账号密码，选着一张图片点击提交，就可以看到这些信息发送到后台并被接收后的结果！
